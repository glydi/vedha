## How to Redeploy

### Option 1: Manual Redeploy via Dashboard (Recommended for now)

1. Go to https://dashboard.render.com
2. Select **vedha-backend** service
3. Click **"Redeploy"** button at the top-right
4. Select **main** branch (if not already selected)
5. Click **"Redeploy"** to start

### Option 2: Automatic Redeploy (Already in Progress)

Since we pushed to GitHub, Render may automatically redeploy. Check the **Events** tab to see if a new build started.

## Troubleshooting

### Backend won't start
1. Check Logs for Java/Spring errors
2. Verify database credentials are correct
3. Check if PORT environment variable is set to 10000

### CORS errors on frontend
1. Update `CORS_ALLOWED_ORIGINS` in Backend environment variables
2. Set to: `https://vedha-frontend.onrender.com`
3. Redeploy backend

### Frontend not loading
1. Check build logs - npm install might fail
2. Verify `package.json` has all dependencies
3. Check `vite.config.js` is correct

### Database connection errors
1. Verify IP address: `npvwzgnwprxt.db.dbaas.dev:30731`
2. Check credentials in Environment variables
3. Ensure database `PwKFWx` exists
